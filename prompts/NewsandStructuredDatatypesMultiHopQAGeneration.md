# News-type and Chart-type Multi-Hop Question-Answer Generation Protocol

### Objective

Multiple high-quality Q&A pairs are generated by selecting multiple key points from two sets of sequential key points. The goal is to build a practical knowledge base that enables users to understand important or complex data, specialized information. One set of keypoints is from video and one set of keypoints is from text, with consistent subject matter and different content.

### Methodology
The process is: Sequential Keypoints Selection --> Question Generation --> Answer Generation

### 1. Sequential Keypoints Selection
The number of keypoints selected at a time is between 1 and 4 from the video, and those from the text must be greater than 1 and not greater than the number of videos.

### 2. Question Generation
The questions include, but are not limited to, the following patterns:
Causal Synthesis (Why):
To understand the reasons, motivations, or underlying factors behind a visually observed event. Use questions like "What factors led to..." or "What was the official justification for...".
Elaborative Synthesis (What):
To enrich a general event, object, or person with specific, concrete details (like performance data, names, technical specs, or financial terms). Pinpoint the specific subject (e.g., a new device, a signed document) and ask for its precise attributes. Be direct and specific in your query.
Comparative Synthesis (How it Compares):
To compare or contrast subjects, often by relating a visual from the video to analytical data from the text that might concern a competitor, a benchmark, or a previous state.Clearly state the two items being compared and the specific basis for the comparison. Use phrases like "How does X's performance compare to Y's...", "What is the key difference between...".
Consequential Synthesis (What's Next or So What?):
To connect an action or event with its future outcomes, next steps, broader significance, or projected impact as described. Describe the event and ask about its results, implications, or the next steps in the process. Use phrases like "Following the [event], what are the expected impacts on...", "What procedural next steps are required after...".

- Specific objects or entities must be explicitly identified. When referring to specific entities (e.g., people, organizations, locations), their full names or affiliations must be specified as clearly as possible (e.g., use “U.S. Congress” instead of “Congress”).
- The question needs all the selected key points for a complete answer, and none of them is missing. 
- Avoid trivial or universal questions. The questions should be relevant.
- Don't simply string two questions together or write two unrelated questions in the same sentence. Instead, look for connections between the questions.
Incorrect example: What items were placed at the memorial on Tithebarn Road in Southport, and how did the families of the murder victims and Liverpool faith leaders respond to the violent protests?

### 3. QA Generation
3 core principles:
Retrievability: Does the question contain enough context to be found precisely?
Human-like quality: Do the questions make sense and resemble questions that a real person would ask?
Answer Quality: Is the answer accurate, comprehensive and helpful?

- Before generating a Q\&A pair, verify that the selected video keypoint provides unique, essential information that the text keypoint lacks, and which is critical for forming the answer. If the video key point is merely illustrative or decorative (e.g., "a person is talking," "an object is on a table") and does not add a factual cornerstone to the answer, you must discard this pairing and select a different set of keypoints.
- - A QA pair should be independent and understandable without any context.
- The Answer must directly and completely address the question, using the factual information contained within the source keypoint.
- Generate specific, closed-ended QA pairs.
- Avoid source attribution. Do not mention "the video", "the narrator", or "the keypoint". Treat the information as factual knowledge.
Prohibited phrases:  
    Camera type: “The close-up shows...,” “In the frame...”  
    Production type: “Video summary...,” “The narrator mentions...”  
    Explanatory type: “This indicates...”(Unless the key point contains explanatory information)
- Ensure the language is natural, as if a user is asking for help and an expert is providing a direct answer.
- Generate as many as question-answer pairs from the provided keypoints.
- The output must be a JSON file and formatted to match the example without any interpretation or code markup.
